# Security policy

Security fixes are applied to the latest released minor version. Older pre-1.0
versions may require upgrading.

Do not open a public issue for a vulnerability. Use GitHub private vulnerability
reporting and include the affected version, a minimal reproduction, impact, and
suggested mitigation. Maintainers will acknowledge complete reports within five
business days and coordinate disclosure after a fix is available.

Relevant boundaries include external URI construction, network error
classification, and denial-of-service behavior from untrusted string inputs.
